# people-daily-researching-ds（无DS后缀者为Codex等主流Agent适用)

低资源版《人民日报》电子版标题筛选 skill。适合只能操作浏览器、不能稳定批量开标签页、不能可靠读写本地文件的 agent。

## 适用场景

用于按日期查看《人民日报》电子版版面，收集各版文章标题和正文链接，并根据筛选词判断哪些标题可能与目标行业相关。

默认筛选行业：

- 旅游
- 美妆
- 日化

## 调用示例

```text
/people-daily-researching-ds 2026年7月4日 默认
```

或：

```text
/people-daily-researching-ds 20260704 旅游 美妆 日化
```

如果用户已经写了 `默认`，agent 不要再次询问筛选词，直接使用默认行业词。

## 核心流程

1. 规范化日期，例如 `20260704`。
2. 打开第一页版面：
   `https://paper.people.com.cn/rmrb/pc/layout/YYYYMM/DD/node_01.html`
3. 收集当天版面列表和版面 URL。
4. 对版面名称做宽松排除，只跳过明显无关版面。
5. 逐个打开保留版面，只读取文章标题和正文 URL。
6. 只根据标题做宽松相关性判断。
7. 输出标题超链接表格。

## 重要规则

- 不要打开正文页。
- 不要打开 PDF。
- 不要研究源码、隐藏内容或 PDF URL。
- 正文 URL 只用于把 `文章标题` 做成超链接。
- 判断困难但不明显无关的标题也要列出，标为 `标题相关性待核验`。
- 如果工具输出截断或 agent 重复同一计划，立即停止继续导航，输出已收集结果。

## 输出格式

优先输出 HTML 表格；如果环境不方便生成或渲染 HTML，直接在聊天中输出同样内容的表格即可，不要为了 HTML 卡住。

表格字段：

- `版面`
- `文章标题`
- `PDF链接`
- `匹配依据`

要求：

- `文章标题` 必须做成超链接。
- `PDF链接` 默认写 `未打开PDF`。
- 不要包含 `摘要` 列。
- 只有用户明确要求原始 URL 时，才额外加入 `链接` 列。

## 防卡住提示

如果 agent 开始反复思考下一步，直接使用下面提示收束：

```text
停止继续思考和遍历。用户已回复“默认”，直接使用默认筛选词：旅游、美妆、日化。

不要再查看源码、不要打开PDF、不要逐个点正文。只使用当前已拿到的版面文章标题和正文链接。按标题宽松判断相关性：明显相关列为 high，可能相关列为 medium，难判断但不明显无关列为 uncertain。

请立即输出表格，字段为：版面、文章标题、PDF链接、匹配依据。文章标题必须做成可点击超链接；PDF链接统一写“未打开PDF”。不要包含摘要列。
```
